Description

Context

This is part of the authentication system (EPIC-1). The User model was created in TASK-101, and password hashing is being implemented in TASK-98.

Objective

Create a service that generates JWT tokens for authenticated users. Tokens should include user ID, email, and role claims with appropriate expiration.

Files to Modify

  • src/services/auth.rs (create)
  • src/models/token.rs (create)
  • Cargo.toml (add jsonwebtoken dependency)

Acceptance Criteria

  • ✅ Generate JWT with configurable expiration
  • ⏳ Include user claims (id, email, role)
  • ⏳ Sign with RS256 algorithm
  • ⏳ Unit tests cover happy path and errors

Design Philosophy
  • Speed First: Every interaction should feel instant. No loading spinners for common actions.
  • Keyboard-Driven: Power users should rarely need to touch the mouse.
  • Progressive Disclosure: Show simple by default, reveal complexity only when needed.
  • Agent-Aware: The UI should clearly distinguish agent activity from human activity.
  • Mobile-Friendly: Essential features work perfectly on mobile devices.
