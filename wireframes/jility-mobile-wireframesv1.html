Mobile Design Principles
  • Touch Targets: All interactive elements are 44x44px minimum for easy tapping
  • Bottom Navigation: Primary navigation at thumb-reach for one-handed use
  • FAB (Floating Action Button): Quick ticket creation always accessible
  • Swipe Gestures: Swipe cards between columns, swipe to delete/archive
  • Pull to Refresh: Update ticket lists by pulling down
  • Action Sheets: Native-feeling bottom sheets for contextual actions
  • Horizontal Scroll: Board view scrolls horizontally through columns
  • Safe Areas: Respects iOS notch and Android navigation bars
  • Offline Support: Core features work without internet (future)
