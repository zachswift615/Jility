Project Switcher Button
← Click to open project switcher modal (see Wireframe #0)
Design Details:
  • Project Icon: Shows first 2 letters of project name with project color
  • Button Style: Uses btn-secondary (consistent with other secondary actions)
  • Hover State: Subtle background change (#eef1f5)
  • Active State: Opens modal overlay (not a dropdown menu)
  • Keyboard: Can also trigger with ⌘P or Ctrl+P shortcut
💡 Alternative: Inline Dropdown
Instead of a modal, you could use an inline dropdown that appears below the button. This would be faster for quick switching between 2-3 recent projects.

Design Philosophy
  • Speed First: Every interaction should feel instant. No loading spinners for common actions.
  • Keyboard-Driven: Power users should rarely need to touch the mouse.
  • Progressive Disclosure: Show simple by default, reveal complexity only when needed.
  • Agent-Aware: The UI should clearly distinguish agent activity from human activity.
  • Mobile-Friendly: Essential features work perfectly on mobile devices.
